# Contributing

GLASS Padel repositories are maintained by the core team; collaborators are invited per project.

## Workflow

- Branch from the default branch using `kind/short-kebab` (for example `fix/booking-timezone`, `feat/venue-dashboard`).
- One concern per commit, one feature per pull request.
- Open a pull request. Direct pushes to `main` are not permitted.
- Write commit messages in British English, imperative mood, with the subject under 72 characters.

## Standards

- Clear names, simple structure, no shortcuts.
- Validate input at every trust boundary; never commit secrets.
- Tests for non-trivial logic.
